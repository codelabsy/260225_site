## 1. 시스템 전체 흐름도 (System Overview)

```mermaid
graph TB
    subgraph CLIENT["클라이언트 (웹 브라우저)"]
        UI[HTML/TailwindCSS/shadcn UI]
        JS[Vanilla JS - AJAX]
    end

    subgraph SERVER["서버 (PHP)"]
        AUTH[인증 미들웨어]
        ROUTER[라우터 - API 엔드포인트]

        subgraph MODULES["비즈니스 모듈"]
            M_AUTH[인증 모듈]
            M_ERP[ERP 매출 모듈]
            M_SHOP[쇼핑 DB 모듈]
            M_PLACE[플레이스 DB 모듈]
            M_DASH[대시보드 모듈]
            M_STAT[통계 모듈]
        end

        subgraph CORE["코어 레이어"]
            ORM[ORM 모델]
            LOG[활동 로그]
            PERM[권한 검증]
            EXCEL[엑셀 처리]
        end
    end

    subgraph DB["데이터베이스"]
        SQLITE[(SQLite3 - crm.sqlite)]
    end

    UI -->|사용자 입력| JS
    JS -->|AJAX 요청| AUTH
    AUTH -->|세션 검증| ROUTER
    ROUTER --> MODULES
    MODULES --> CORE
    CORE --> ORM
    ORM -->|PDO| SQLITE
    LOG -->|기록| SQLITE
    SQLITE -->|결과| ORM
    ORM -->|JSON 응답| JS
    JS -->|DOM 업데이트| UI
```

## 2. 사용자 인증 흐름도 (Authentication Flow)

```mermaid
flowchart TD
    START([사용자 접속]) --> CHECK_SESSION{세션 존재?}

    CHECK_SESSION -->|Yes| VALIDATE_SESSION{세션 유효?}
    CHECK_SESSION -->|No| LOGIN_PAGE[로그인 페이지 표시]

    VALIDATE_SESSION -->|Yes| CHECK_ROLE{역할 확인}
    VALIDATE_SESSION -->|No / 만료| DESTROY_SESSION[세션 파기]
    DESTROY_SESSION --> LOGIN_PAGE

    LOGIN_PAGE --> INPUT_CRED[아이디/비밀번호 입력]
    INPUT_CRED --> VALIDATE_INPUT{입력값 검증}
    VALIDATE_INPUT -->|실패| ERROR_MSG[에러 메시지 표시]
    ERROR_MSG --> INPUT_CRED

    VALIDATE_INPUT -->|성공| QUERY_USER[DB에서 사용자 조회]
    QUERY_USER --> USER_EXISTS{사용자 존재?}
    USER_EXISTS -->|No| ERROR_MSG

    USER_EXISTS -->|Yes| CHECK_ACTIVE{계정 활성화?}
    CHECK_ACTIVE -->|No| ERROR_INACTIVE[비활성 계정 안내]
    ERROR_INACTIVE --> INPUT_CRED

    CHECK_ACTIVE -->|Yes| VERIFY_PW{비밀번호 검증}
    VERIFY_PW -->|실패| ERROR_MSG

    VERIFY_PW -->|성공| CREATE_SESSION[세션 생성]
    CREATE_SESSION --> REGENERATE[session_regenerate_id]
    REGENERATE --> LOG_LOGIN[로그인 로그 기록]
    LOG_LOGIN --> CHECK_ROLE

    CHECK_ROLE -->|ADMIN| ADMIN_DASH[관리자 대시보드]
    CHECK_ROLE -->|EMPLOYEE| EMP_DASH[직원 대시보드]

    style START fill:#e1f5fe
    style ADMIN_DASH fill:#c8e6c9
    style EMP_DASH fill:#c8e6c9
    style ERROR_MSG fill:#ffcdd2
    style ERROR_INACTIVE fill:#ffcdd2
```

## 3. ERP 매출 관리 흐름도

### 3.1 업체 등록 흐름

```mermaid
flowchart TD
    START([업체 등록 시작]) --> CHECK_PERM{관리자 권한?}
    CHECK_PERM -->|No| DENY[접근 거부]
    CHECK_PERM -->|Yes| SHOW_FORM[등록 폼 표시]

    SHOW_FORM --> INPUT_BASIC[기본 정보 입력]
    INPUT_BASIC --> INPUT_DETAIL[상세 정보 입력]
    INPUT_DETAIL --> INPUT_AMOUNT[금액 정보 입력]

    INPUT_AMOUNT --> AUTO_CALC[자동 계산 실행]
    AUTO_CALC --> CALC_VAT["부가세 = 결제금액 / 11"]
    CALC_VAT --> CALC_MARGIN["순마진 = 결제금액 - 실행비 - 부가세"]
    CALC_MARGIN --> DISPLAY_CALC[계산 결과 표시]

    DISPLAY_CALC --> SUBMIT{제출?}
    SUBMIT -->|No| INPUT_BASIC

    SUBMIT -->|Yes| VALIDATE[서버 입력값 검증]
    VALIDATE -->|실패| SHOW_ERROR[검증 오류 표시]
    SHOW_ERROR --> INPUT_BASIC

    VALIDATE -->|성공| BEGIN_TX[트랜잭션 시작]
    BEGIN_TX --> SAVE_COMPANY[companies 테이블 저장]
    SAVE_COMPANY --> TRIGGER_CALC[트리거: 순마진 재계산]
    TRIGGER_CALC --> SAVE_DETAIL[company_details 테이블 저장]
    SAVE_DETAIL --> LOG_ACTIVITY[활동 로그 기록]
    LOG_ACTIVITY --> COMMIT[트랜잭션 커밋]
    COMMIT --> SUCCESS[등록 완료 알림]

    style START fill:#e1f5fe
    style SUCCESS fill:#c8e6c9
    style DENY fill:#ffcdd2
```

## 4. CRM 네이버 쇼핑 DB 흐름도

### 4.1 엑셀 업로드 흐름

```mermaid
flowchart TD
    START([엑셀 업로드 시작]) --> SELECT_FILE[엑셀 파일 선택]
    SELECT_FILE --> CHECK_FORMAT{파일 형식 검증}

    CHECK_FORMAT -->|실패: .xlsx/.xls/.csv 아님| ERROR_FORMAT[파일 형식 오류]
    ERROR_FORMAT --> SELECT_FILE

    CHECK_FORMAT -->|성공| UPLOAD[서버 업로드]
    UPLOAD --> PARSE[엑셀 파싱]
    PARSE --> COUNT_TOTAL["전체 건수 카운트"]

    COUNT_TOTAL --> LOOP_START{다음 행 있음?}
    LOOP_START -->|No| SHOW_RESULT[업로드 결과 표시]

    LOOP_START -->|Yes| READ_ROW[행 데이터 읽기]
    READ_ROW --> EXTRACT_PHONE[전화번호 추출]
    EXTRACT_PHONE --> CHECK_DUP{전화번호 중복?}

    CHECK_DUP -->|중복| MARK_DUP[중복 표시 + 카운트]
    MARK_DUP --> LOOP_START

    CHECK_DUP -->|고유| INSERT_DB[shopping_db INSERT]
    INSERT_DB --> SET_USER["user_id = 업로드자"]
    SET_USER --> SET_STATUS["status = '안함'"]
    SET_STATUS --> INC_SUCCESS[성공 카운트 +1]
    INC_SUCCESS --> LOOP_START

    SHOW_RESULT --> SAVE_HISTORY[upload_histories 저장]
    SAVE_HISTORY --> LOG_ACTIVITY[활동 로그 기록]
    LOG_ACTIVITY --> DONE([업로드 완료])

    subgraph RESULT["업로드 결과 표시"]
        R1["전체: N건"]
        R2["성공: N건"]
        R3["중복: N건"]
        R4["실패: N건"]
    end

    style START fill:#e1f5fe
    style DONE fill:#c8e6c9
```

## 14. 권한 검증 흐름 (모든 API 공통)

```mermaid
flowchart TD

    CHECK_ACTIVE -->|Yes| CHECK_CSRF{CSRF 토큰 유효?}

    IS_ADMIN -->|No| RETURN_403
    IS_ADMIN -->|Yes| PROCESS[비즈니스 로직 실행]

    CHECK_PERM -->|인증만 필요| CHECK_SCOPE{데이터 범위 확인}

    ALL_DATA --> PROCESS
    OWN_DATA --> CHECK_OWNER{본인 배정 데이터?}
    CHECK_OWNER -->|No| RETURN_403
    CHECK_OWNER -->|Yes| PROCESS

    PROCESS --> RETURN_200["200 OK<br/>성공 응답"]
```

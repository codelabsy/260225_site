# ERP + CRM 통합 시스템 요구정의서 (SRS)

> **문서 버전**: v1.0
> **작성일**: 2026-02-25
> **프로젝트명**: ERP + CRM 통합 관리 시스템
> **기술스택**: HTML, PHP, SQLite, TailwindCSS, shadcn/ui, Vanilla JS
> **제약사항**: Composer 미사용, SSH 미사용

---

## 1. 프로젝트 개요

### 1.1 목적
매출 관리(ERP), 네이버 쇼핑 DB 관리, 네이버 플레이스 DB 관리, 외부 광고 DB 관리를 하나의 웹 기반 시스템에서 통합 운영하는 것을 목표로 한다.

### 1.2 핵심 목표
| 번호 | 목표 | 설명 |
|------|------|------|
| G-01 | 매출/DB/콜/통계 통합 관리 | 모든 영업 데이터를 단일 플랫폼에서 관리 |
| G-02 | 권한 분리 | 관리자/직원 역할 기반 접근 제어 |
| G-03 | DB 배분·회수 시스템 | 관리자가 직원에게 DB를 배분하고 회수하는 워크플로우 |
| G-04 | 실시간 상태/히스토리 관리 | 모든 변경사항의 실시간 추적 및 이력 보존 |
| G-05 | 웹 기반 시스템 | PC 최적화 필수, 웹 브라우저에서 동작 |

### 1.3 기술 제약사항
| 항목 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 마크업 | HTML5 | 시맨틱 마크업 |
| 백엔드 언어 | PHP (순수 PHP) | Composer 미사용, 프레임워크 미사용 |
| 데이터베이스 | SQLite3 | 파일 기반 DB, 별도 서버 불필요 |
| CSS 프레임워크 | TailwindCSS (CDN) | CDN 방식 또는 빌드 없이 사용 |
| UI 컴포넌트 | shadcn/ui 스타일 | shadcn 디자인 패턴 참고 커스텀 구현 |
| JavaScript | Vanilla JS | jQuery/React 등 프레임워크 미사용 |
| 서버 접근 | SSH 미사용 | FTP 또는 웹 패널 기반 배포 |

## 2. 사용자 역할 및 권한 정의

### 2.1 역할 분류

| 역할 코드 | 역할명 | 설명 |
|-----------|--------|------|
| ROLE_ADMIN | 최상위 관리자 | 시스템 전체 관리 권한 보유 |
| ROLE_EMPLOYEE | 직원 | 배정된 DB에 대한 제한적 권한 보유 |

## 3. 기능 요구사항 (Functional Requirements)

### 3.2 [AUTH] 인증/계정 모듈

#### FR-AUTH-001: 로그인
| 항목 | 내용 |
|------|------|
| 설명 | 사용자가 아이디/비밀번호로 시스템에 로그인 |
| 입력 | 아이디, 비밀번호 |
| 처리 | 1. 입력값 검증 2. DB 조회 3. 비밀번호 검증(password_hash) 4. 세션 생성 5. 로그인 로그 기록 |
| 출력 | 성공: 대시보드 리다이렉트 / 실패: 에러 메시지 |
| 권한 | 모든 사용자 |

#### FR-AUTH-002: 로그아웃
| 항목 | 내용 |
|------|------|
| 설명 | 세션 파기 및 로그인 페이지 리다이렉트 |
| 처리 | 1. 세션 파기 2. 로그아웃 로그 기록 3. 로그인 페이지 리다이렉트 |
| 권한 | 모든 로그인 사용자 |

#### FR-AUTH-003: 직원 계정 생성
| 항목 | 내용 |
|------|------|
| 설명 | 관리자가 직원 계정을 생성 |
| 입력 | 아이디, 비밀번호, 이름, 직급, 연락처, 이메일(선택) |
| 처리 | 1. 아이디 중복 검사 2. 비밀번호 해시 처리 3. 계정 생성 4. 생성 로그 기록 |
| 검증 | 아이디 4~20자 영문+숫자, 비밀번호 8자 이상 |

| 항목 | 내용 |
|------|------|
| 설명 | 관리자가 직원 계정 정보를 수정 |
| 입력 | 이름, 직급, 연락처, 이메일, 비밀번호(선택), 활성화 상태 |
| 처리 | 1. 대상 계정 조회 2. 정보 업데이트 3. 수정 로그 기록 |
| 권한 | ROLE_ADMIN |

#### FR-AUTH-005: 직원 계정 삭제(비활성화)
| 항목 | 내용 |
|------|------|
| 설명 | 관리자가 직원 계정을 비활성화 (소프트 삭제) |
| 처리 | 1. 계정 비활성화 처리 2. 배정 DB 자동 회수 3. 삭제 로그 기록 |
| 제약 | 물리적 삭제 금지, 비활성화만 처리. 배정된 DB는 미배분 상태로 변경 |
| 권한 | ROLE_ADMIN |

#### FR-AUTH-006: 세션 관리
| 항목 | 내용 |
|------|------|
| 설명 | PHP 세션 기반 인증 상태 유지 |
| 처리 | 1. 페이지 로드마다 세션 유효성 검증 2. 타임아웃 시 자동 로그아웃 3. 역할 기반 접근 제어 |

### 3.3 [ERP] 매출 관리 모듈

#### FR-ERP-001: 업체 등록
| 항목 | 내용 |
|------|------|
| 설명 | 신규 거래 업체를 시스템에 등록 |
| 입력 필드 (기본) | 담당자(직원 선택), 등록일, 상품명, 상호명, 결제금액, 계산서 발행금액, 실행비, 직급 |
| 입력 필드 (상세) | 계약 시작일, 계약 종료일, 업체명, 대표자명, 연락처, 결제구분, 사업자번호, 작업 키워드, 작업 내용, 이메일, 네이버 계정 정보, 실행비 입력란 |
| 자동 계산 | 순마진 = 결제금액 - 실행비 - (결제금액 기준 부가세) |
| 처리 | 1. 입력값 검증 2. 자동 계산 필드 처리 3. DB 저장 4. 등록 로그 기록 |
| 권한 | ROLE_ADMIN |

#### FR-ERP-002: 업체 수정
| 항목 | 내용 |
|------|------|
| 설명 | 등록된 업체 정보를 수정 |
| 처리 | 1. 대상 업체 조회 2. 변경 필드 업데이트 3. 자동 계산 필드 재계산 4. 변경 로그 기록 |
| 제약 | 수정 전후 데이터 히스토리 보존 |
| 권한 | ROLE_ADMIN |

#### FR-ERP-003: 업체 상세 조회
| 항목 | 내용 |
|------|------|
| 설명 | 업체의 전체 정보 및 히스토리 조회 |
| 출력 | 기본정보, 상세정보, 일자별 메모 히스토리, 순마진 자동계산값 |
| 권한 | ROLE_ADMIN, ROLE_EMPLOYEE(본인 담당만) |

#### FR-ERP-004: 업체 목록 조회
| 항목 | 내용 |
|------|------|
| 설명 | 업체 목록을 필터/검색/정렬 기능과 함께 조회 |
| 필터 | 기간(년/월/일), 담당자, 결제구분, 상태 |
| 검색 | 상호명, 대표자명, 연락처, 사업자번호 (부분 검색 지원) |
| 정렬 | 등록일, 결제금액, 순마진 기준 오름차순/내림차순 |
| 페이징 | 50 / 100 / 300 / 500 / 1000건 선택 표시 |
| 권한 | ROLE_ADMIN: 전체 / ROLE_EMPLOYEE: 본인 담당만 |

#### FR-ERP-005: 계약 이월(연장)
| 항목 | 내용 |
|------|------|
| 설명 | 기존 업체 정보를 복사하여 신규 계약으로 등록 |
| 처리 | 1. "이월" 버튼 클릭 2. 기존 업체 정보 전체 복사 3. 계약기간 필드 초기화(수정 가능 상태) 4. 사용자가 계약기간 수정 5. 신규 레코드로 저장 |
| 복사 항목 | 업체명, 대표자명, 연락처, 사업자번호, 결제구분, 작업키워드, 작업내용, 이메일, 네이버 계정정보, 담당자 |
| 초기화 항목 | 계약 시작일, 계약 종료일, 결제금액, 실행비, 메모 |
| 제약 | 원본 업체 데이터는 그대로 유지, 신규 레코드로 별도 생성 |
| 권한 | ROLE_ADMIN |

#### FR-ERP-006: 일자별 메모 (히스토리 누적)
| 항목 | 내용 |
|------|------|
| 설명 | 업체별 일자별 메모를 히스토리 방식으로 누적 저장 |
| 처리 | 1. 메모 입력 2. 작성자/작성시간 자동 기록 3. 기존 메모 유지(덮어쓰기 금지) 4. 시간 역순 표시 |
| 제약 | 기존 메모 수정/삭제 불가, 새 메모만 추가 가능 |
| 권한 | ROLE_ADMIN, ROLE_EMPLOYEE(본인 담당) |

#### FR-ERP-007: 매출 자동 계산
| 항목 | 내용 |
|------|------|
| 설명 | 매출 관련 수치를 자동으로 계산 |
| 계산식 | |
| | 총 매출 = SUM(결제금액) |
| | 총 실행비 = SUM(실행비) |
| | 총 부가세 = SUM(결제금액 기준 부가세) |
| | 총 순마진 = 총 매출 - 총 실행비 - 총 부가세 |
| | 개별 순마진 = 결제금액 - 실행비 - 부가세 |
| 트리거 | 결제금액 또는 실행비 입력/수정 시 실시간 재계산 |

### 3.4 [CRM_SHOPPING] 네이버 쇼핑 DB 관리 모듈

## 8. 용어 정의

| 용어 | 정의 |
|------|------|
| DB | 영업용 데이터베이스 레코드 (고객/업체 정보) |
| 배분 | 관리자가 DB를 직원에게 할당하는 행위 |
| 회수 | 관리자가 직원에게 배분된 DB를 되돌려 받는 행위 |
| 이월 | 기존 계약 정보를 복사하여 신규 계약으로 등록하는 행위 |
| 콜 | 영업 전화 처리 행위 |
| 순마진 | 매출 - 실행비 - 부가세 |
| 부가세 | 결제금액 기준 부가가치세 (결제금액 / 11) |
| 실행비 | 실제 집행된 비용 |
| 기관총 | 외부 DB 추출 도구명 |
| 히스토리 | 변경 이력의 누적 기록 |

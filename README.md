# ERP + CRM 통합 관리 시스템

매출 관리(ERP) + 네이버 쇼핑 DB + 네이버 플레이스 DB + 외부 광고 DB를 하나의 웹 기반 시스템에서 통합 운영하는 관리 시스템

## 기술 스택

| 구분 | 기술 |
|------|------|
| Backend | PHP (순수 PHP, No Framework, No Composer) |
| Database | SQLite3 |
| Frontend | HTML5, Vanilla JS |
| CSS | TailwindCSS (CDN) |
| UI | shadcn/ui 스타일 커스텀 컴포넌트 |
| 배포 | FTP 기반 (SSH 미사용) |

## 주요 기능

### ERP - 매출 관리
- 업체 등록/수정/상세 조회
- 결제금액, 실행비, 부가세, 순마진 자동 계산
- 계약 이월(연장) 기능
- 일자별 메모 히스토리 누적

### CRM - 네이버 쇼핑 DB
- 엑셀 업로드 (전화번호 중복 체크)
- 상태 관리: 부재 / 재통 / 가망 / 계약완료 / 안함
- DB 배분/회수
- 상태 변경 히스토리 자동 기록

### CRM - 네이버 플레이스 DB
- 관리자 직접 등록
- 상태 관리: 부재 / 재통 / 가망 / 거절 / 계약완료
- DB 배분/회수
- 상태 변경 히스토리 자동 기록

### 대시보드 / 통계
- 관리자: 전체 매출 집계, 직원별 비교, 목표 달성률, 그래프 시각화
- 직원: 본인 매출/순마진 조회, 월별 그래프
- 엑셀 다운로드 (관리자 전용)

### 계정 / 권한
- 최상위 관리자: 전체 데이터 관리, 직원 계정 관리, 엑셀 다운로드
- 직원: 본인 배정 DB만 조회/수정, 다운로드 불가

## 디렉토리 구조

```
crm_new/
├── config/          # DB 연결, 상수, 세션 설정
├── core/            # Database, Auth, Permission, Logger 등
├── models/          # ORM 모델 클래스
├── api/             # REST API 엔드포인트
├── pages/           # 페이지 PHP 파일
├── templates/       # 공통 헤더/푸터/컴포넌트
├── assets/          # CSS, JS 정적 파일
├── data/            # SQLite DB 파일
├── uploads/         # 엑셀 업로드 임시 저장
├── logs/            # 활동 로그
└── docs/            # 기획 문서
```
